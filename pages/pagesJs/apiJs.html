O que seria uma API?

A sigla API vem de Application Programming Interface (Interface de Programação de Aplicações) é um praticamente uma aplicação que permite que diferentes outras aplicações se comuniquem entre si na internet. Existem diversas aplicações para os mais diversos tipos de trabalhos e funcionalidades que podem ser instaladas em vários projetos.Geralmente são utilizadas para acessar e manipular dados de um servidor, utilizando formatos como JSON ou XML para a troca de informações. Elas são fundamentais para que haja a integração de sistemas, permitindo não só fazer isso de mais eficaz, mas também de maneira padronizada

Apis são utilizadas praticamente em tudo que utilizamos no dia a dia, tanto para utilizar as redes sociais, tanto para comprar em uma loja online e para transferencias bancárias como o Paypal!

A seguir teremos algumas opções de utilizar as API'S em nossos projetos:

FETCH API

O método do FETCH é uma API moderna do javascript que permite fazer requisições HTTP de forma assíncrona. Esse método retorna um Promise que resolve para o objeto Response, representando a resposta da requisição. A seguir iremos explorar com detalhes de como o fetch funciona:

Obs : Será utilizado para fins educacionais a API via cep! Confira a documentação e o site oficial no link abaixo:

https://viacep.com.br/

  • FETCH

    • O comando fetch(url, options) é utilizado para buscar e requisitar uma API

  • THEN

    • O comando then() é utilizado lidar com a resolução de uma Promise. Ele é executado assim que a operação é executada com sucesso

    Geralmente o comando then é utilizado para receber os valores vindos de uma operação bem sucedida ou para lidar com os dados processados. Assim podendo capturar os pontos de interesse da API requisitada

    Exemplo :

    THEN

    Obs: Dentro deste exemplo em específico, nós temos o primeiro then para processar a resposta da API, e o segundo then para a lidar com os dados que foram processados, nesse caso o conteúdo em JSON no console.log

    Adendo : O cep é o número que sempre está aqui "...ws/CEP/json..."

  • CATCH

    • O método catch() é utilizado para retornar e tratar erros que ocorrem durante a execução de uma Promise, sempre sendo executada assim que um Promise nos retornar "REJEITADO". Isso pode incluir diversos tipos de erros, como de conexão ou acesso aos dados.

    Exemplo :

    CATCH

    Obs: Dentro deste exemplo em específico, nós forçamos a API dar um problema passando a ela um cep "0", assim ela no final voltando a requisição como "REJEITADA", e além disso o nosso console.log mostrará nossa mensagem de erro

    Recomendo fazer uma rápida leitura na parte de Promise para ter o conhecimento de como essas requisições vão funcionar por completo!
